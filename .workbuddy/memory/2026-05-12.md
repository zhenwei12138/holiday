# 2026-05-12 工作日志

## 大中华区节假日 GitHub Actions 自动化项目

**任务**：创建一个 GitHub Actions 自动化项目，定期采集大陆、台湾、香港三地法定节假日并输出统一格式 JSON。

**项目结构**：
```
/Users/zhenwei.liu/WorkBuddy/2026-05-12-task-7/
├── .github/workflows/update-holidays.yml   # GitHub Actions workflow
├── scripts/
│   ├── models.py          # 共享数据结构与工具函数
│   ├── fetch_mainland.py  # 大陆采集（国务院 gov.cn + 静态回退）
│   ├── fetch_taiwan.py    # 台湾采集（DGPA CSV + 动态链接发现）
│   ├── fetch_hongkong.py  # 香港采集（1823.gov.hk iCal JSON）
│   └── fetch_all.py       # 主入口，合并三地数据
├── types/holidays.d.ts    # TypeScript 类型定义
├── data/                  # 输出数据（已生成 2025/2026 年数据）
├── requirements.txt
└── README.md
```

**数据源**：
- 大陆：中国政府网搜索国务院通知，已知直链 + 静态回退数据（2025/2026）
- 台湾：人事行政总处 DGPA CSV（真实 API，已验证）；民国115年链接 dgpa/files/202506/a52331bd...csv
- 香港：1823.gov.hk/common/ical/tc.json（繁中 JSON），字段结构：dtstart[0]=日期字符串, summary=字符串

**已验证**：本地运行 fetch_all.py 成功生成 2025/2026 两年数据

**触发策略（已升级为高频探测模式）**：
- **探测期**：11月1日~次年1月15日，每3天自动运行一次，持续搜索国务院通知
- **确认期**：1月下旬 + 2月初各一次，确保数据完整
- workflow_dispatch 支持手动指定年份、地区、强制提交
- 每次运行自动判断大陆数据来源质量（官方正文 vs 回退数据）并写入 Summary

**升级原因**：国务院发布通知时间不固定（10月下旬~12月初），单次11月1日运行可能抓不到。改为高频探测后最多延迟3天即可获取真实数据。

**输出格式**：符合 HolidayCalendarDay TypeScript 接口，含 region/date/year/type/name/isHoliday/isWorkday/adjust/remark
